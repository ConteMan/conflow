# Spec 014：业务配置编辑器

> 状态：待实现  
> 依赖：Spec 004、006、007、012、013

## 目标

让 PM 通过业务对象完成广告位、频控、开关和环境 unit binding 的新增、修改、复制、删除与保存，不接触长 JSON。

## 结构前提（Spec 012 评审收口，2026-07-10）

- 实体浏览用按实体类型分 tab 的全宽数据表；广告位编辑用独立详情页（分区表单 + 环境×平台绑定矩阵）。
- 频控策略等被引用实体的编辑用右侧抽屉，抽屉内必须展示受影响广告位清单，保持数据表上下文可见。
- 「基线值 / 环境覆盖 / 线上当前值」用字段级来源 caption 表达（如「通用值 · 未发布（线上为 X）」），不做三列并排。
- 验收 fixture 以 docs/design/ui/prototypes/fixtures.md 的实体命名与规模为准。

## 合同前置

开工前关闭 [`UI-API-005` 至 `UI-API-013`](../design/ui/contract-gaps.md#spec-014-开工门槛)。草稿层级、字段来源、冲突恢复、实体引用、远端值投影、校验 freshness 与共享 fixture 必须先进入后端 Spec 和 OpenAPI；React 不自行定义这些 DTO 或规则。

UI-API-009 于 2026-07-11 由维护者决策降级：014 首版编辑器的字段 caption 不含线上值，待 Spec 008/009 后补充。

## 范围

- 实体列表、筛选、引用计数、风险状态和详情编辑。
- Pack schema 驱动基础字段；高价值复杂字段使用移动广告专用交互。
- 基线值、环境覆盖和 effective value 的来源展示。
- 表单级即时检查与服务端权威诊断。
- 删除保护、被引用对象跳转、复制后生成新稳定 ID。
- revision 冲突时展示本地草稿与服务端当前值，用户选择重载或手动重做。
- 高级区展示 RC key、源映射和只读 JSON 预览。

## API

消费 Spec 003、004、006、007 的 Pack、draft 和 validation 端点。

## 非范围

- 任意 schema 的无代码表单设计器。
- 通用 JSON 编辑作为默认入口。

## 验收

- PM 可完成“新增广告位 → 绑定频控 → 配置 Production unit → 保存 → 校验”。
- 删除被引用频控时显示引用者并阻止误删。
- 环境不可覆盖字段在 GUI 中不可编辑，服务端仍二次校验。
- 20+ 广告位和长名称 fixture 下列表、搜索与编辑可用。
