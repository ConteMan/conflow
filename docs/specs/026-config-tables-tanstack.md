# Spec 026：配置列表迁移 TanStack Table

> 状态：待实现  
> 依赖：Spec 014（UI 域编辑器）  
> 关联：Issue #51；实施顺序在 #53（描述列）、#54（开关编辑）合并之后，避免同文件冲突

## 目标

四个配置列表（广告位、频控策略、功能开关、广告单元绑定）迁移到无头表格库 TanStack Table，一次性解决列固定、滚动提示、统计位置等长期手搓痛点，并为后续排序、筛选类需求提供内建能力。

## 背景

列表为手工原生 table，列宽、截断、横向滚动提示全靠零散 CSS 补丁，已多次返工（列被裁切、`table-layout: fixed` 破版、滚动条不可见）。项目已用 `@base-ui/react`（无头交互控件），但其不含表格；TanStack Table（`@tanstack/react-table` v8）是 React 无头表格事实标准，样式完全自控、与现有设计体系及 Base UI 无冲突。

## 范围

### 依赖与结构

- 新增依赖 `@tanstack/react-table`（core 无 UI，gzip 约 14 KB）。
- 新建共享表格组件（如 `web/src/components/ui/DataTable.tsx`）：封装 column pinning、滚动阴影、空状态、行点击；四个列表以列定义（`ColumnDef`）方式接入，删除各自手搓的 table 标记与专用 CSS 补丁。

### 交互合同

- **列固定**：首列（key/标识列）pin left，末列（未发布修改 / 操作）pin right；容器可横向滚动时，被固定列内侧渲染阴影作为"还有内容"提示。
- **工具栏**：统计信息（总数 / 筛选命中数）与类型筛选、搜索框、"仅看未发布修改"合并为表格顶部一条工具栏；移除底部统计。
- **排序**：key、类型等可排序列启用客户端排序（默认按 key 升序，与现状一致）。
- **现有行为保持**：行点击进详情/抽屉、筛选语义、"未发布修改"标记逻辑均不变（标记口径由 Spec 024 独立演进）。

### 数据流

纯前端重构：继续消费既有 API 读模型，不改请求与响应合同；客户端排序筛选，数据量级（当前 28 广告位 / 11 频控 / 30 开关）远低于需要虚拟化的阈值。

## 非范围

- 虚拟滚动、分页、服务端排序筛选。
- 列显隐配置、列宽拖拽等表格个性化。
- 校验中心、发布计划等其余页面的表格（后续视效果另行迁移）。

## 验收

- 四个列表迁移完成，视觉与现有设计语言一致（对照 `conflow-ui.pen` 设计稿）。
- 窄视口下横向滚动：首末列保持固定可见，阴影提示正确出现与消失。
- 工具栏在表格顶部展示统计与筛选，底部无统计残留。
- 行点击、筛选、"仅看未发布修改"行为与迁移前一致（e2e 或手工回归清单）。
- `make check` 通过（含 `web-check` 资产同步与 e2e）。
