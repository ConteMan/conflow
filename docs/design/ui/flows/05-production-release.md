# 状态矩阵：Production 发布与回滚入口

## 场景

Plan 审阅完成后执行发布。Staging 等非生产环境走常规确认；Production 走升级的危险确认。发布后进入审计记录与回滚入口（Spec 010/011）。

## 状态矩阵

| 状态 | 触发 | UI 表现 | 出口 |
|---|---|---|---|
| 入口 | Plan 页发布 CTA | 常规环境：单步确认弹层，展示变更摘要；Production：全屏/大尺寸确认流程 | 确认 / 取消 |
| 危险确认 | Production 发布 | ① 变更与风险清单强制可见（不折叠高风险项）② 输入环境 ID 确认 ③ 高风险变更（如删除参数、覆盖条件值边缘）追加确认勾选。确认强度随 Plan 风险等级动态升级 | 执行 / 取消 |
| 加载 | 发布执行中 | 分步进度：预校验 → 提交 → 确认生效；不可关闭但可查看详情；带幂等键，重复点击无副作用 | 成功 / 错误 / 冲突 |
| 正常（发布后） | 发布完成 | 发布记录页：谁、何时、改了什么、目标版本号、回滚入口 | 回滚 / 返回 |
| 空 | 发布历史为空 | 「此环境还没有发布记录」 | 返回 |
| 错误 | 远端拒绝 / 网络失败 | 明确失败发生在哪一步、线上是否已变化；给出重试（幂等）或返回 Plan | 重试 / 返回 |
| 冲突 | 提交时 ETag 失效 | 发布中止，线上未变化；引导重建 Plan | 重建 Plan |
| 成功 | 发布生效 | 成功页 + 审计记录链接；提示「在 <环境> 验证行为」；若来自 Staging 流程，提供「继续到 Production」的显式入口 | 验证 / 下一环境 |
| 危险确认（回滚） | 从发布记录发起回滚 | 展示将恢复到的版本与差异，同级确认强度 | 执行回滚 |

## 关键规则

- **确认强度动态化**：采用「风险等级 → 确认要求」映射，而不是固定输入确认短语。确认强度是项目级配置项（2026-07-10 评审收口）：默认所有 Production 发布都要求输入环境 ID；项目配置可放宽为「低风险 Plan 仅勾选确认」，高风险项的逐项勾选任何配置下都不可关闭。
- **失败必须回答两个问题**：线上现在是什么状态；我下一步做什么。
- **回滚是一等入口**：出现在发布成功页和发布历史，不藏在菜单里。
