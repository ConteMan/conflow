# 状态矩阵：广告位与频控编辑

## 场景

PM 在配置页浏览 20+ 广告位，编辑单个广告位字段，或修改被多个广告位引用的频控策略。规模基准见 [fixtures](../prototypes/fixtures.md)。

## 状态矩阵

| 状态 | 触发 | UI 表现 | 出口 |
|---|---|---|---|
| 入口 | 进入配置页 | 按实体类型组织的列表；默认展示启用状态、类型、频控引用、未发布修改标记 | 编辑 |
| 加载 | 拉取实体列表 | 表格骨架屏，保留列头 | 正常 / 错误 |
| 正常 | 列表载入 | 20+ 行可扫读：支持按类型/启用/有未发布修改过滤；单行进入编辑 | 编辑 / 新建 |
| 空 | Pack 尚无实体 | 空态说明该实体是什么 + 「新建」+ 文档链接；不显示空表格线框 | 新建 |
| 错误（字段） | 保存时字段校验失败 | 错误锚定到字段行内提示；提交按钮列出错误计数并可逐个跳转 | 修复 |
| 错误（系统） | 写入失败 / IO 错误 | 编辑内容保留在表单；错误条给出重试；不清空输入 | 重试 |
| 冲突 | 保存返回 `412`（他处已修改） | 阻断保存；提供「查看最新值」对比当前输入，选择基于最新值重改或放弃 | 重载 / 放弃 |
| 危险确认 | 删除实体 / 删除被引用的频控 | 删除被引用策略时先展示引用者清单，未处理引用不允许确认；普通删除单次确认 | 删除 / 取消 |
| 成功 | 保存草稿成功 | 行内出现「未发布修改」标记；全局出现「有未发布修改」汇总状态 | 继续编辑 / 去校验 |

## 编辑面特有规则

- **基线 vs 环境覆盖**：字段级标识当前值来自「通用值」还是「此环境专属」；改基线时明示影响的环境集合。稳定 ID、placement 不可环境覆盖（Spec 006），UI 直接不提供覆盖入口而不是置灰。
- **频控修改的影响面**：编辑频控策略时展示引用它的广告位清单（不需要 PM 去 Plan 页才发现）。
- **保存 ≠ 发布**：保存只落草稿；按钮文案用「保存修改」，不用「提交」「应用」。

## 文案基调

- 字段名使用 Pack 提供的中文 UI metadata；单位写进标签（「冷却时间（秒）」）。
- 错误必须可行动：「冷却时间需在 0–3600 秒之间」而非「数值非法」。
