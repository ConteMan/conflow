# 状态矩阵：Plan 构建与语义 Diff

## 场景

发布前 PM 查看「本次发布会改什么」：业务侧语义 diff（改了哪些广告位/策略/开关）、目标侧 diff（影响哪些远端参数）、风险与影响面（Spec 008）。

## 状态矩阵

| 状态 | 触发 | UI 表现 | 出口 |
|---|---|---|---|
| 入口 | 校验通过后的主 CTA / 导航 | 进入即自动构建当前环境 Plan | 加载 |
| 加载 | Plan 构建 + 拉取远端当前值 | 骨架屏区分「本地计算」与「读取线上配置」两个阶段 | 正常 / 错误 |
| 正常 | Plan 就绪 | 业务 diff 为主视图：按实体分组的变更卡片（新增/修改/删除/覆盖），每组标注风险；远端参数 diff 在高级区 | 发布 / 返回修改 |
| 空 | 无变更 | 「当前环境没有待发布的修改」；若其他环境有变更给出指引 | 切环境 / 返回 |
| 错误（本地） | 编译失败 | 定位到导致失败的实体，返回修复 | 返回配置 |
| 错误（远端） | 线上配置不可达 | 业务 diff 仍可看；远端对比区显示不可用 + 重试；发布按钮禁用并说明原因 | 重试 |
| 冲突 | 远端 ETag 与上次拉取不一致 | 阻断发布：「线上配置刚被修改」，强制重新构建 Plan 并高亮远端侧变化 | 重建 Plan |
| 危险确认 | 检测到会覆盖线上条件值 | 独立阻断流程：列出冲突条件值，默认不允许继续（硬规则：禁止静默覆盖） | 返回 / 维护者路径 |
| 成功 | Plan 审阅完成 | 「发布到 <环境>」CTA，Production 走升级确认 | 发布流程 |

## 关键规则

- **大 diff 可扫读**：9+ 实体变更时先给数量摘要（「9 个广告位、3 个开关、1 个环境覆盖」），再逐组展开；不默认全展开。
- **业务 diff 与远端 diff 的关系**：业务 diff 是给 PM 的主叙事；远端参数 diff 是证据链，放高级区（逐层展开，而不是并列 tab —— 该假设由两个方向分别验证）。
- **Plan 是快照**：Plan 绑定草稿 revision 与远端 ETag；任一变化即失效，UI 不允许发布过期 Plan。

## 待原型验证

- 变更卡片粒度：按实体一卡 vs 按字段一行。
- 风险标注的位置：卡片角标 vs 汇总区独立清单。
