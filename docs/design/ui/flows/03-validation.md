# 状态矩阵：校验与发布就绪度

## 场景

PM 保存草稿后，需要知道当前环境的配置是否可发布：字段规则、引用完整性、环境绑定完整性和发布就绪度（Spec 007）。校验是显式动作 + 后台增量提示的组合。

## 状态矩阵

| 状态 | 触发 | UI 表现 | 出口 |
|---|---|---|---|
| 入口 | 「有未发布修改」状态 / 校验页 | 显示上次校验时间与结果摘要；草稿改动后标记「结果可能过期」 | 运行校验 |
| 加载 | 校验运行中 | 摘要区进度指示；已有结果保留并置灰，不清空 | 正常 / 错误 |
| 正常（通过） | 无阻断问题 | 明确「可发布到 <环境>」+ 警告类问题折叠展示 | 去 Plan |
| 正常（有问题） | 存在诊断 | 问题按实体分组列表：严重度、锚点、可行动文案；点击跳转到对应字段 | 修复 → 重跑 |
| 空 | 无未发布修改 | 「没有需要校验的修改」+ 上次发布信息 | 返回配置 |
| 错误 | 校验引擎自身失败 | 系统错误条 + 重试；区别于「校验发现问题」 | 重试 |
| 冲突 | 校验期间草稿又被修改 | 结果标记「基于旧版本」，提供一键重跑 | 重跑 |
| 危险确认 | —（校验只读，无危险操作） | — | — |
| 成功 | 校验通过 | 就绪徽标进入全局状态；「查看发布计划」成为主 CTA | 去 Plan |

## 关键规则

- **问题分级**：阻断（不可发布）/ 警告（可发布但提示）/ 建议。只有阻断改变就绪度。
- **锚点强制**：每条诊断必须能跳到实体或字段；没有锚点的诊断视为校验引擎缺陷。
- **按环境**：就绪度是环境属性；Staging 通过不代表 Production 通过（绑定缺失常只发生在一个环境）。

## 待原型验证

- 大量诊断（8+ 条跨多实体）时按实体分组 vs 按严重度分组的扫读效率。
