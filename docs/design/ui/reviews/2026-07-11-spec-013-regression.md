# 评审记录：Spec 013 实现的设计回归

> 日期：2026-07-11
> 方法：`conflow serve` + 真实浏览器走查（macOS Chrome），对照 `conflow-ui.pen` 的 Spec 013 画板与 [screen-inventory](../screen-inventory.md)。fixture 为 `conflow init` 默认 workspace（photo-editor，2 环境）。
> 结论：结构与合同语义大体忠实，**412 冲突对照是亮点**；发现 1 个阻断级断点缺陷和若干顶栏/文案偏差，需修复后再验收关闭。

## 验证通过

- 顶栏壳、组合选择器、面包屑、统计卡网格、环境列表布局与设计一致。
- 「未发布修改」全局状态槽存在且如实标注「尚未接入」（`UI-API-003` 降级正确，未伪造 dirty 状态）。
- Production 持续风险标识：顶部危险色条 + 顶栏变调 + 常驻横幅，由服务端 `Environment.kind` 驱动（`UI-API-001` 运行时正确），切换环境无确认弹窗，且不只依赖颜色。
- 环境管理页与设计画板高度一致：表格 + 右侧编辑面板，环境 ID / 类型锁定并带说明。
- **保存 `412` 冲突弹窗完全符合设计语义**：「项目清单已被其他操作修改」+ 服务端 revision + 「我的修改 / 服务端当前值」双栏对照 +「保留我的输入 / 重新加载当前值」+ 明示不自动覆盖（`UI-API-002` 运行时正确）。
- 窄视口导航折叠为汉堡菜单（机制存在，断点错误见缺陷 1）。

## 观察（非缺陷，待后续 Spec 确认）

- 统计卡内容为可用数据降级（环境数/配置来源/Provider/最近发布 vs 设计的 配置实体/未发布修改/校验状态/远端连接），合同占位期合理；004/007 接入后按设计收敛。
- SPA 与 API 同源托管：服务停止后硬刷新是浏览器错误页，设计的「无法连接到 Conflow 本地服务」页只覆盖「SPA 已加载、后续 API 失败」场景；且概览数据有客户端缓存，服务停止后切页无感知。stale 检测/重连语义建议在 013 E2E 中明确。
- 环境选择器为原生 select（设计为自定义组合选择器），013 阶段可接受，实现 014 前建议切换为设计系统组件。
