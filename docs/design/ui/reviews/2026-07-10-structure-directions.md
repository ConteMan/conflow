## 探索的两个方向

| | 方向 A：顶栏 + 数据表 + 抽屉 | 方向 B：侧栏主从 + 详情页 |
|---|---|---|
| 全局导航 | 顶栏水平导航，项目/环境组合选择器在顶栏 | 左侧栏垂直导航，项目/环境卡片在侧栏顶部 |
| 实体浏览 | 全宽数据表（7 列） | 中栏列表（2 行/项）+ 右侧详情 |
| 实体编辑 | 右侧抽屉覆盖表格 | 独立详情页（分区表单） |
| 校验入口 | 常驻底部状态栏 | 侧栏导航项 + badge |
| 语义 diff | 分组卡片 + 高级区折叠（远端 diff 独立区） | 逐层展开树（变更 → 受影响广告位 → 远端参数） |
| Production 确认 | 计划页上的 modal | 独立步骤页（审阅 → 确认 → 执行） |
| 屏 | A1、A2 | B1、B2、B2b |

两方向共享的已验证约定：Production 持续变调（红条/红徽标）而非切换弹窗；「未发布修改」全局徽标；字段级「已修改 · 原值/线上值」caption；被引用频控编辑时展示受影响广告位清单。

## 用真实规模数据得到的观察

- **24 行扫读**：A 的数据表明显优于 B 的两行式列表——列对齐让「哪些禁用、哪些绑定不完整」一眼可见；B 的列表在 1160 高度只放得下 20 项。
- **长名称**（`compress_complete_interstitial_legacy_campaign`，44 字符）：A 表格 330px 列与 B 的 296px 列表都会截断；两方向都需要 tooltip 或中键展开，不构成方向差异。
- **编辑表单容量**：广告位详情（基本信息/加载行为/频控/环境绑定 3 环境 × 2 平台）在 B 的 748px 详情页从容；塞进 A 的 510px 抽屉会挤（本轮抽屉只放了 4 字段的频控编辑已接近舒适上限）。
- **频控修改的影响可见性**：A 的抽屉能同时看到底层表格（受影响行仍在视野）；B 编辑频控时上下文完全切走。这是抽屉真正的优势场景。
- **大 diff（14 处修改 / 12 个远端参数）**：B 的逐层展开树让「业务变更 → 受影响广告位 → 最终写入参数」证据链连续，PM 不需要在两个区域间来回对照；A 的「高级区折叠」把远端 diff 与业务 diff 切成两个语境。
- **Production 确认**：B2b 独立步骤页比 A2 modal 有更强的减速感和更充足的风险清单空间；modal 更容易被当作普通确认框习惯性点掉。
- **窄屏**：A 的数据表无法优雅降级；N1/N2 实际是 B 结构的窄屏投影（列表 → 详情）。若壳采用顶栏，窄屏导航需单独设计（汉堡菜单）。

## 混合结构（已采纳，2026-07-10）

1. **壳用 A 的全局顶栏**：项目/环境常驻最高视觉层级，Production 变调面积大（整条顶栏 + 红条），窄屏折叠代价可接受。
2. **实体浏览用 A 的数据表**：24+ 行的扫读与过滤效率是 PM 高频动作。
3. **实体编辑用 B 的详情页**（表单空间、三值 caption、绑定矩阵），**被引用实体（频控策略）的编辑用抽屉**——利用其「受影响行可见」优势。已由维护者确认。
4. **Plan 采用 B 的逐层展开树**，放弃「业务 diff / 远端 diff 并列 tab」的原始假设。
5. **Production 发布用 B2b 的独立步骤页**，非 modal。

## 放弃项及原因

- **「基线值 / 环境覆盖 / 线上值」三列并排展示**：认知负担重、占宽（DESIGN.md §8 问题 3）。改为字段级 caption（「通用值 · 未发布（线上 #57 为 5000）」）+ Plan 页作完整对照。
- **切换到 Production 时弹确认框**：切换是读操作；减速留给写操作，持续视觉状态代替一次性弹窗（见 [01-project-environment-switch.md](../flows/01-project-environment-switch.md)）。
- **语义 diff / 远端 diff 并列 tab**：证据链被切断；A2 中实现为「高级区折叠」已是弱化版，最终被 B2 的逐层展开替代。
- **固定确认短语（如输入 "yes-i-know"）**：改为输入环境 ID + 高风险项逐项勾选，确认强度随 Plan 风险等级动态升级（见 [05-production-release.md](../flows/05-production-release.md)）。
